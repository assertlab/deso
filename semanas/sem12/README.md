- [ ] Aplicar um checklist avançado de code review que vai além de erros de sintaxe: funcionalidade, testes, legibilidade, arquitetura e documentação

## Por que esta semana importa para o Sprint 4

A Semana 12 é a primeira semana do **Sprint 4 — o sprint final**. Esse sprint tem uma natureza diferente dos anteriores: o foco não é adicionar funcionalidades novas, é **estabilizar e tornar o produto sustentável**.

No contexto da Sinfonia, estamos no movimento **Ensaio** — fase final antes da Ressonância. A Ressonância (Semanas 14–15) é quando o produto sai das mãos da equipe e entra nas mãos do stakeholder para uso real. Um produto sem documentação adequada chega a essa fase incompleto — não porque falta código, mas porque o conhecimento sobre o código ficou preso na cabeça de quem o escreveu.

O code review cruzado desta semana não é um exercício didático: é um ensaio da perspectiva externa. A outra equipe que vai revisar o código de vocês é um proxy do stakeholder, do professor, ou do próximo desenvolvedor que assumir o projeto. O que eles encontrarem vai revelar exatamente o que a documentação e a qualidade do código ainda precisam comunicar.

**A entrega formal ao stakeholder está a 2 semanas. Esta é a última oportunidade de ajustar o que o produto comunica sobre a equipe que o construiu.**
